What is Variable

Variable

Variable is a "named storage container" for storing a data.
Variables allow you to store data that can be referenced and manipulated throughout your program.
They can hold different types of data, such as numbers, strings, objects, arrays, and more.
We can use variables to store goodies, visitors, and other data.

Declaring a variable

To use a variable in JavaScript, you need to first of all create it more accurately, we call this declaring the variable.
Alright so, before you can declare a variable, you need to pick any of the three keywords which includes :
Keywords for declaring variable ::::

  • var :::: This is the keyword for declaring a variable traditional way. It has function scope and can be re-declared and updated.
    Back when JavaScript was first created, this was the only way to declare variables. The design of var is confusing and error-prone.
    Due to this a new keyword was introduced to correct the problems encountered with the var.

  • const :::: This keyword was also introduced in ES6, it is used to declare block-scoped constant variable.
    Once assigned, their values cannot be changed (though the contents of objects or arrays can be modified).

Since the var keyword is no more in use, in view of that in this our lesson we're only going to make use of the let and const keywords.

Differences Between The Old var And The New let Javascript Keywords Used For The Declaration Of Variables :::::

Meaning Of Key-Points var let
Scope :::: refers to the accessibility or visibility of variables in certain parts of your code.
And we have three types ("Global scope, Function scope and Block scope."). 		Has function scope. If declared inside a function, it is only accessible within that function. If declared outside any function, it has global scope. Has block scope. It is only accessible within the nearest enclosing block (e.g., within {} braces).
Hoisting :::: is a JavaScript mechanism where variables and function declarations are moved to the top of their containing scope during the compilation phase. Variables declared with var are hoisted to the top of their containing function or global context. This means they can be referenced before their declaration, but will be undefined. Variables declared with let are also hoisted, but they cannot be accessed until the line of code where they are declared is reached. This creates a "temporal dead zone" from the start of the block until the declaration.
Re-declaration :::: refers to declaring a variable with the same name in the same scope. Allows re-declaration of the same variable within the same scope without throwing an error. Does not allow re-declaration of the same variable in the same scope. Attempting to do so will result in a SyntaxError.
Global object :::: is a special object that serves as the top-level scope for all JavaScript code running in a particular environment (like a browser or Node.js). When declared globally, it becomes a property of the global object (e.g., window in browsers). When declared globally, it does not create a property on the global object.


Reasons For Using let Over var

  • Block Scope ::: The block-scoping nature of let allows for better management of variable visibility and lifecycle, reducing potential errors caused by variables leaking outside their intended scope.
  • Avoiding Hoisting Issues ::: The temporal dead zone associated with let helps prevent issues that arise from hoisting, making code easier to understand and maintain.
  • Preventing Re-declarations ::: By disallowing re-declarations in the same scope, let reduces the risk of accidental overwrites and bugs in larger codebases.
  • Modern JavaScript Practices ::: As JavaScript has evolved, let (and const, which is also block-scoped) has become the preferred way to declare variables due to its clearer semantics and reduced risk of errors.

Variable Naming In JavaScript

Rules For Naming JavaScript Variable ::::

Variable naming refers to the conventions and rules you should follow when creating variable names in programming.
Proper naming is crucial for code readability, maintainability, and clarity.
There are limitations on variable names in JavaScript and these include ::::

  • The name must contain only letters ( a-z or A-Z), digits ( 0-9 ), or the symbols dollar signs ( $ ) and underscores ( _ ) .
  • The first character must not be a digit.
  • Don't use underscores at the start of variable names this is used in certain JavaScript constructs to mean specific things, so may get confusing.
  • Don't use numbers at the start of variables. This isn't allowed and causes an error.
  • A safe convention to stick to is lower camel case, where you stick together multiple words, using lower case for the whole first word and then capitalize subsequent words. We've been using this for our variable names in the article so far.
    That is: words go one after another, each word except first starting with a capital letter: myVeryLongName.
  • Make variable names intuitive, so they describe the data they contain. Don't just use single letters/numbers, or big long phrases.
  • Variables are case sensitive — so myage is a different variable from myAge.
  • One last point: you also need to avoid using JavaScript reserved words as your variable names by this, we mean the words that make up the actual syntax of JavaScript!
    So, you can't use words like var, function, let, and for as variable names. Browsers recognize them as different code items, and so you'll get errors.

Constants in JavaScript

Constant is a variable that cannot be changed or reassigned once it's declared.
An attempt to do so would cause an error.
When a programmer is sure that a variable will never change, they can declare it with constant to guarantee and communicate that fact to everyone.
Though constant are like variables except that you must initialize them when you declare them and also you can't assign them a new value after you've initialized them.
For Example ::::

// using let you can declare a variable without initializing it:
let count;
// If you try to do this using const you will see an error:

Similarly, with let you can initialize a variable, and then assign it a new value (this is also called reassigning the variable):
For Example ::::

count = 2
// If you try to do this using const you will see an error:
const count = 1
count = 2 // this code will produce an error.

Although a constant in JavaScript must always name the same value, you can change the content of the value that it names.
This isn't a useful distinction for simple types like numbers or booleans, but consider an object.
For Example ::::

const bird = { species: "Kestrel" };
document.write(bird.species);

If you can't do as much with const as you can with let, why would you prefer to use it rather than let?

Use const when you can, and use let when you have to.

JavaScript Hoisting ::::

Hoisting is a fundamental concept in JavaScript that refers to the way variable, classes and function declarations are processed by the interpreter during the compilation phase of the code execution.
It essentially means that JavaScript "hoists" or lifts declarations to the top of their containing scope before the code is executed, allowing you to use variables and functions before they are actually declared in the code.


In colloquial terms, any of the following behaviors may be regarded as hoisting :::
  • Variable Hoisting ::: When you declare a variable using var, the declaration is hoisted to the top of its enclosing function or global scope. However, only the declaration is hoisted, not the initialization.
    For Example ::::
    //This code outputs undefined
    document.write(x);
    let x = 5;
    //This code outputs 5
    document.write(x);

    In this case, the first document.write outputs undefined result because the declaration let x; is hoisted to the top, but the assignment x = 5; happens later.

  • Function Hoisting :::: Function declarations are also hoisted. This means you can call a function before it appears to be defined in the code.
    For Example :::: 
     // The below code outputs "Hello!"
                greet( );
function greet( ) { 
  document.write("Hello!"); 
       }

    Here, the function greet can be called before its declaration because the entire function definition is hoisted.
    And can also be called after too.

  • Let and Const :::: Variables declared with let and const are also hoisted, but they are not initialized. Accessing them before their declaration results in a ReferenceError. This is known as the "temporal dead zone."
    For Example ::::
    // ReferenceError: The code below cannot access 'y' before initialization
    document.write( y );
    let y = 30;
  • Function Expressions :::: If a function is defined as a function expression (e.g., assigned to a variable), it is not hoisted in the same way as function declarations.
    For Example ::::
    // TypeError: greet is not a function
    let greetings = function ( ) { document.write("Hello!"); }

    In this case, greet is hoisted as a variable declaration but not initialized as a function until the line where it's defined.


Introduction To Scope In JavaScript

In JavaScript, variable scope refers to the accessibility or visibility of variables within different parts of your code. That is the various places you have put you variable codes in your entire codebase.
Understanding variable scope is crucial for managing how data is accessed and modified throughout your application.

Types Of Scopes In JavaScript ::::

  • Global Scope :::: Variables declared outside any function or block are in the global scope.
    These variables can be accessed from anywhere in the code, including inside functions.

    In JavaScript, global scope is the widest scope available. Variables declared in global scope are accessible from anywhere in your code, whether it's inside functions, conditional statements, loops, or other blocks of code.

    You can think of global scope as the "public square" of your program, where everyone can see and access what's going on.

    To illustrate let's consider an analogy ::::

    Imagine your JavaScript program as a small town, and the global scope as the town's central square.
    Everything you declare in global scope is like putting up a billboard in the square for everyone to see.
    Any function or code block within your program can read and modify these global variables.

    Variables Declared in Global Scope ::::

    Variables declared in global scope are typically defined outside of any functions or code blocks. 

    let globalVariable = "I'm in global scope";
function myFunction() {
/" This function can access globalVariable */
document.write(globalVariable);
}
myFunction();
    Here, globalVariable is declared in global scope, and myFunction can access and use it directly.

    Accessing global variables is straightforward. You can use them within functions or any part of your code without any special effort.
    It's like having a public notice board in the town square where everyone can read and post messages.

    Potential Issues With Global Scope ::::

    While global scope can be quite convenient, it comes with potential downsides.
    Since global variables are accessible from anywhere, they are susceptible to unintended changes and conflicts.

    For instance, if multiple parts of your code modify the same global variable, it can lead to unexpected behaviors and bugs that are hard to track down.
    Moreover, global variables can make your code less modular and organized. If everything is in the town square, it becomes challenging to manage and isolate different aspects of your program.

  • Function/Local Scope :::: Variables declared within a function using var, let and constant are scoped to that function.

    Local scope in JavaScript is like a private room within a building it's an enclosed space where variables are only accessible from within that specific room.
    When you declare a variable in local scope, it is limited in visibility to the block of code, function, or conditional statement in which it is defined.
    Variables in local scope are protected from interference or modification by code outside their scope, providing a level of isolation.

    Variables in local scope are typically declared within functions, conditional statements, loops, or other code blocks.
    These variables are, in essence, "local" to that block of code, and they cannot be directly accessed from outside it.

    Consider the following analogy ::::

    If a local scope is like a private room, a variable declared within that scope is akin to an object or piece of furniture placed inside that room.
    Others can't directly interact with it from outside – they would need permission, like a key, to enter the room and access the variable.

    Variables defined inside a function are not accessible from outside the function and variables declared with var, let and const are quite similar when declared inside a function.

    We continue by declaring a new variable named result and we assigned the Inside Condition. This new declaration creates a block-scoped variable that only exists within the curly braces {} of the if statement. It shadows (or hides) the outer result variable declared earlier in the function.
    Alright we now as the browser to display the contents of the variable result so meaning when we run the code we are expecting the browser to display to us two things those are Before Condition and Inside Condition on our page.
    But you will realize that when we run the code the browser only displayed to us Before Condition instead of the browser displaying what we planned to achieve.
    The reason why this happened is that since document.write(result) is outside the block of the if statement, it refers to the outer result, which contains the value "Before Condition".

    Block scope is a powerful tool for managing variable visibility and preventing unintended variable clashes within specific code blocks.
    It enhances code modularity and helps you write more maintainable and predictable JavaScript code. Understanding block scope is essential for efficient and organized coding.




When To Use Different Types Of Scope
  • Global Scope :::: Use global scope only for variables that genuinely need to be accessed throughout your application.
    Global variables should be rare exceptions and carefully managed.
  • Local and Block Scope :::: Embrace local and block scope to keep variables isolated and contained.
    Use local scope within functions and block scope for variables within specific code blocks like loops and conditional statements.
